# API Specification Audit & Review
**Project Name:** EduPulse AI  
**Role Perspective:** Principal .NET API Architect & School ERP Domain Expert  
**Status:** Audit Completed  

---

## 1. Executive Summary

This audit reviews the proposed RESTful API contracts in `07_API_Specification.md`. While the specification covers basic operations, several critical business workflows, reports, settings, and student lookup APIs necessary for a viable Indian school launch are missing. 

To ensure the MVP is complete and launch-ready, this review identifies the missing endpoints and details their request/response schemas, validation checks, and database authorization parameters.

## 2. Missing CRUD APIs

### 2.1. Update Subject Configuration
* **Endpoint:** `PUT /api/subjects/{subjectId}`
* **Method:** `PUT`
* **Request Model:**
  ```json
  {
    "name": "Social Studies",
    "subjectCode": "SOC101"
  }
  ```
* **Response Model (204 No Content):** Empty.
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Checks for name/code duplication within the tenant context.
  * Restricts editing of `subjectCode` if historical exam marks are already mapped to this subject.

### 2.2. Soft Delete Subject
* **Endpoint:** `DELETE /api/subjects/{subjectId}`
* **Method:** `DELETE`
* **Response Model (204 No Content):** Empty.
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Enforces soft-delete isolation (`IsDeleted = 1`).
  * Blocks delete request (returns `ERR_SUBJECT_IN_USE`) if active class sections are currently mapped to this subject.

### 2.3. Soft Delete Fee Line Item
* **Endpoint:** `DELETE /api/fees/line-items/{feeLineItemId}`
* **Method:** `DELETE`
* **Response Model (204 No Content):** Empty.
* **Authorization Rules:** `RequiresRole(SchoolAdmin, Accountant)`
* **Business Rules:**
  * Blocks deletions if invoices have already been generated referencing this line item. Returns `ERR_FEE_ITEM_INvoiced` (HTTP 400).

## 3. Missing Workflow APIs

### 3.1. Request Student Transfer Certificate (TC)
* **Endpoint:** `POST /api/students/{studentId}/tc/request`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "reason": "Family relocation to another state."
  }
  ```
* **Response Model (200 OK):**
  ```json
  {
    "studentId": "e1a9c321-4e10-4820-bbf5-894c244243b7",
    "status": "Transferred",
    "isPendingTC": true
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Flags student profile status as ready for transition checks. Does not generate a TC number.

### 3.2. Issue Transfer Certificate & Archive Student
* **Endpoint:** `POST /api/students/{studentId}/tc/issue`
* **Method:** `POST`
* **Response Model (200 OK):**
  ```json
  {
    "studentId": "e1a9c321-4e10-4820-bbf5-894c244243b7",
    "tcNumber": "TC/2026/0014",
    "issueDate": "2026-06-18",
    "status": "Transferred"
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * **Zero Outstanding Dues Rule:** Queries `StudentInvoices` where `StudentId = @StudentId`. If `BalanceAmount > 0`, blocks issue and returns `ERR_OUTSTANDING_FEE_DUES` (HTTP 400).
  * Automatically assigns a formatted serial number utilizing the school settings prefix (e.g. `TC/2026/0014`).
  * Transitions student status permanently to `Transferred`, closes active classroom registries, and locks user login access for the parent profile.

### 3.3. Attendance Log Unlock Override
* **Endpoint:** `POST /api/attendance/unlock`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "classSectionId": "5c9c3210-4e10-4820-bbf5-894c244243b3",
    "date": "2026-06-18"
  }
  ```
* **Response Model (200 OK):**
  ```json
  {
    "classSectionId": "5c9c3210-4e10-4820-bbf5-894c244243b3",
    "unlockedUntil": "2026-06-18T14:30:00Z"
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Bypasses the 09:30 AM attendance lock state for a specific section, allowing the class teacher to resubmit attendance. Sets an automatic lock re-evaluation window of 30 minutes.

## 4. Missing Promotion & Rollover APIs

### 4.1. Validate Academic Year Rollover Preconditions
* **Endpoint:** `POST /api/academic-years/rollover/validate`
* **Method:** `POST`
* **Response Model (200 OK):**
  ```json
  {
    "isValid": false,
    "preconditions": {
      "allExamsPublished": true,
      "unsubmittedAttendanceDays": 0,
      "activeInvoicesUncollectedValue": 45000.00
    }
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Scans active year registries. Warnings are flagged if pending exams exist in draft status. Returns a count of unpaid invoice assets to be carried over.

### 4.2. Execute Year-End Promotion & Rollover
* **Endpoint:** `POST /api/academic-years/rollover/execute`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "targetAcademicYearId": "d3b9c321-4e10-4820-bbf5-894c244243a1",
    "promotions": [
      {
        "studentId": "e1a9c321-4e10-4820-bbf5-894c244243b7",
        "action": "Promote", -- Promote / Repeat
        "targetClassSectionId": "7f9c3210-4e10-4820-bbf5-894c244243c4"
      }
    ]
  }
  ```
* **Response Model (200 OK):**
  ```json
  {
    "processedCount": 140,
    "errorsCount": 0,
    "activeYearSwitched": true
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Runs inside an explicit database transaction block.
  * Batch inserts new mappings into `StudentClassHistory`.
  * Computes unpaid invoice ledger balances and rolls them over as opening balances in the new term ledger.
  * Updates `TenantSettings.ActiveAcademicYearId` to `targetAcademicYearId`.

## 10. Admission Workflow APIs

### 10.1. Submit Admission Application (Admissions CRM Stage)
* **Endpoint:** `POST /api/admissions`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "firstName": "Amit",
    "lastName": "Kumar",
    "dateOfBirth": "2019-08-10",
    "gender": "Male",
    "parentPhone": "9876543209",
    "parentEmail": "amit.parent@email.com",
    "parentFirstName": "Rajesh",
    "parentLastName": "Kumar",
    "targetClassId": "1a9c3210-4e10-4820-bbf5-894c244243b1"
  }
  ```
* **Response Model (201 Created):**
  ```json
  {
    "applicationId": "b1b9c320-4e10-4820-bbf5-894c244243e8",
    "firstName": "Amit",
    "status": "Applied"
  }
  ```
* **Authorization Rules:** `AllowAnonymous` or `RequiresRole(SchoolAdmin)` (allows public registrations or front desk inputs)
* **Business Rules:**
  * Creates a record in the `Students` table with status set to `Applied`. Sibling or target parent checks are parsed by matching the `parentPhone` string.

### 10.2. List Admission Applications
* **Endpoint:** `GET /api/admissions`
* **Method:** `GET`
* **Request Query Parameters:**
  * `pageIndex` (default 1)
  * `pageSize` (default 20)
* **Response Model (200 OK):**
  ```json
  {
    "items": [
      {
        "applicationId": "b1b9c320-4e10-4820-bbf5-894c244243e8",
        "studentName": "Amit Kumar",
        "parentName": "Rajesh Kumar",
        "parentPhone": "9876543209",
        "targetClass": "Class 1",
        "status": "Applied"
      }
    ],
    "totalCount": 8
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`

### 10.3. Approve Admission Application
* **Endpoint:** `POST /api/admissions/{applicationId}/approve`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "admissionNo": "ADM2026105",
    "classSectionId": "5c9c3210-4e10-4820-bbf5-894c244243b3",
    "rollNo": 15
  }
  ```
* **Response Model (200 OK):**
  ```json
  {
    "studentId": "b1b9c320-4e10-4820-bbf5-894c244243e8",
    "admissionNo": "ADM2026105",
    "status": "Admitted"
  }
  ```
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Runs inside a SQL transaction scope.
  * Checks roll number availability in section.
  * Transitions student status to `Admitted`.
  * Creates user login profiles for parents.
  * Populates current year entries inside `StudentClassHistory` using Dapper parameters.

### 10.4. Reject/Archive Admission Application
* **Endpoint:** `POST /api/admissions/{applicationId}/reject`
* **Method:** `POST`
* **Request Model:**
  ```json
  {
    "rejectionReason": "Verification documents missing."
  }
  ```
* **Response Model (204 No Content):** Empty.
* **Authorization Rules:** `RequiresRole(SchoolAdmin)`
* **Business Rules:**
  * Transitions student status to `Archived` (terminal rejected state) and records the rejection reason inside `StudentStatusHistory`.
